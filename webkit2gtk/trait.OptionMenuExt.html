fn activate_item(&self, index: u32)

Activates the OptionMenuItem at index in self. Activating an item changes the value of the element making the item the active one. You are expected to close the menu with OptionMenuExt::close after activating an item, calling this function again will have no effect.

Feature: v2_18

index

the index of the item

fn close(&self)

Request to close a OptionMenu. This emits WebKitOptionMenu::close signal. This function should always be called to notify WebKit that the associated menu has been closed. If the menu is closed and neither OptionMenuExt::select_item nor OptionMenuExt::activate_item have been called, the element value remains unchanged.

Feature: v2_18

fn select_item(&self, index: u32)

Selects the OptionMenuItem at index in self. Selecting an item changes the text shown by the combo button, but it doesn't change the value of the element. You need to explicitly activate the item with OptionMenuExt::select_item or close the menu with OptionMenuExt::close in which case the currently selected item will be activated.

Feature: v2_18
